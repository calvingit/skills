## 代码简化

在保持功能完整的前提下，改善代码表达，使其更清晰、一致、易读、易维护。该技能侧重轻量整理和表达优化，而不是结构性重构。

## 何时使用

当出现以下情况时使用本技能：

- 新写或刚修改的代码可以更清晰、更直接
- 代码存在轻度冗余、命名不清、嵌套偏深或表达绕弯
- 需要统一风格、简化写法、减少阅读负担
- 用户希望“简化这段代码”“整理一下写法”“让代码更清楚”

## 核心原则

1. **保持行为不变**：只改写法，不改功能、输出和对外行为。
2. **优先清晰**：明确、易读的代码优于紧凑、炫技或过度压缩的写法。
3. **遵循现有风格**：与项目已有的命名、组织方式和错误处理习惯保持一致。
4. **只做轻量调整**：聚焦局部表达优化，不主动做模块拆分、职责迁移或大范围结构调整。
5. **默认看近期改动**：优先处理当前会话中最近修改或接触过的代码。

## 常见适用操作

- 简化条件表达式和控制流
- 减少不必要的嵌套和重复判断
- 改善变量、函数、类型或常量命名
- 删除显而易见的冗余注释
- 合并局部重复逻辑或去掉多余中间层
- 将绕弯的写法改成更直接的表达

## 不做的事

- 不改变模块边界或整体架构
- 不主动拆分类、模块或大函数
- 不引入额外设计模式或抽象层级
- 不把轻量整理扩展成完整重构

## 结果标准

- 代码更短但不晦涩
- 逻辑更直观，阅读成本更低
- 风格更统一，命名更准确
- 所有原有行为保持不变
